# Cloud Storage Security Hardening using Google Cloud Platform

## Overview
This project demonstrates the implementation of cybersecurity principles using Google Cloud Security Command Center (SCC). A misconfigured cloud storage bucket was identified, analyzed, remediated, and verified using compliance standards.

## Objectives
- Detect vulnerabilities using SCC
- Analyze misconfigured access controls
- Remediate using IAM policies
- Verify using CIS compliance reports

## Tools Used
- Google Cloud Platform (GCP)
- Security Command Center (SCC)
- Cloud Storage
- IAM (Identity and Access Management)

## Implementation Steps
1. Detected vulnerability using SCC (Public Bucket ACL)
2. Analyzed risk of public access
3. Removed `allUsers` permission
4. Enabled Uniform Bucket-Level Access
5. Verified using CIS compliance report

## Cybersecurity Principles Applied
- Confidentiality
- Access Control
- Monitoring
- Compliance Validation
- Cryptographic Security (AES-256, TLS)

## Results
- Public access removed
- IAM-based control enforced
- Compliance achieved (Findings = 0)
